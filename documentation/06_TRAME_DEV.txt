// TRAME DU CODE - Dream Island //

**Paramètres généraux**

- Taille de l’île : 30 colonnes × 14 lignes → 420 cases
- Taille d’une case / tile : 16×16 px

---

**1️⃣ Création de la carte**

*Objectif :* Créer la grille 2D représentant l’île, gérer placement et suppression des bâtiments.

**Tâches détaillées :**

1. Créer la classe `Island` :
   * Attributs : largeur, hauteur, grille 2D
   * Méthodes : `placer_batiment()`, `supprimer_batiment()`, `case_libre()`

2. Gérer les **contraintes de placement** :

   * Vérifier les cases adjacentes pour les bâtiments interdits à proximité. (VOIR DOC : 03_BATIMENTS_CARACTERISTIQUES)
   * Si placement impossible → renvoyer notification à l’utilisateur.

3. Stocker la position des bâtiments pour sauvegarde JSON.
4. Tester le placement et la suppression → aucune case chevauchée, contraintes respectées.

**Référence documents :** `03_BATIMENTS.txt`, `01_GAMEPLAY.txt`

---
 
**2️⃣ Définition et gestion des bâtiments ** 

*Objectif :* Créer tous les bâtiments avec leurs effets et règles de placement.

**Tâches détaillées :**

1. Créer la classe `Building` :

   * Attributs : nom, catégorie, coût, effets sur 💵/☣️/🌱/😊/👥, niveau requis, capacité si nécessaire, liste de bâtiments interdits à côté.

2. Créer tous les bâtiments du tableau fourni (Maison, Immeuble, Villa, Usine, Parc, etc.).
3. Ajouter logique pour bâtiments spéciaux : Bidonville, Décharge 
4. Tester que chaque bâtiment applique correctement ses effets et respecte les contraintes spatiales.

**Référence documents :** `03_BATIMENTS.txt`

---

**3️⃣ Gestion des indicateurs en temps réel** 

*Objectif :* Mettre à jour 💵, ☣️, 🌱, 😊, 👥 à chaque tick (ex: 1 sec).

**Tâches détaillées :**

1. Créer la classe `Game` :

   * Boucle principale et tick régulier
   * Mise à jour des indicateurs selon bâtiments et événements

2. Implémenter **formules de calcul** :

   * Population → logement + bonheur
   * Bonheur → influence sur population et économie
   * Pollution / biodiversité → influence bâtiments et événements

3. Gérer interactions indicateurs (ex : pollution trop élevée → baisse bonheur).
4. Tester que les indicateurs se recalculent correctement après chaque action.

**Référence documents :** `01_GAMEPLAY.txt`

---

**4️⃣ Système d’événements aléatoires** 

*Objectif :* Générer des événements positifs, négatifs ou neutres.

**Tâches détaillées :**

1. Créer la classe `Event` : nom, type, effets sur indicateurs, durée.
2. Définir fréquence et conditions de déclenchement :

   * Fréquence : toutes les 5 minutes
   * Conditions : pollution > seuil, manque de logements, etc.
3. Faire remonter les événements via **notifications** dans l’interface.
4. Tester impact sur indicateurs et interactions cumulées.

**Référence documents :** `01_GAMEPLAY.txt`

---

**5️⃣ Interface utilisateur Pyxel **

*Objectif :* Afficher carte, bâtiments, indicateurs et boutons.

**Tâches détaillées :**

1. Créer la classe `Interface`.
2. Affichage carte : chaque case = 16×16 px, sprite correspondant au bâtiment.
3. Barre latérale gauche :

   * Afficher indicateurs 💵, ☣️, 🌱, 😊, 👥
   * Boutons : Info, Boutique, Poubelle

4. Fenêtres contextuelles :

   * Infos bâtiment sélectionné
   * Notifications événements
   * Boutons construire/détruire

5. Feedback immédiat : changements d’indicateurs apparaissent instantanément.
6. Tester toutes les interactions : clic bâtiment, boutons, suppression, achat.

**Référence documents :** `01_GAMEPLAY.txt`, `03_BATIMENTS.txt`

---

**6️⃣ Boutique et achat / vente **

**Tâches détaillées :**

1. Affichage de tous les bâtiments disponibles.
2. Vérifier argent et niveau requis → griser les bâtiments non accessibles.
3. Appliquer effets sur indicateurs après achat.
4. Mise à jour interface après transaction.

**Référence documents :** `03_BATIMENTS.txt`

---

**7️⃣ Sauvegarde et chargement JSON **

**Tâches détaillées :**

1. Créer classe `SaveLoad`.
2. Sauvegarder :

   * Liste bâtiments et positions
   * Indicateurs actuels
   * Niveau / progression joueur
   * Événements actifs
3. Charger JSON pour reprendre la partie.
4. Tester plusieurs sauvegardes / chargements → tout restauré correctement.

**Référence documents :** `01_GAMEPLAY.txt`, `04_SYSTEME.txt`

---

**8️⃣ Système de niveaux et progression**

**Tâches détaillées :**

1. Définir paliers de progression (population, argent, bonheur).
2. Débloquer bâtiments et fonctionnalités selon niveau.
3. Afficher alertes dans barre latérale si nouveaux bâtiments disponibles.
4. Tester que chaque palier se déclenche correctement.

**Référence documents :** `03_BATIMENTS.txt`

---

Pour terminer le projet il faut : 

Dans le code Jeu :

Il faut gérer ici le système de "tick" et donc créer une fonction qui permet d'actualiser l'économie (à l'aide des indicateurs). Elle doit en gros parcourir le self.batiments_places et pour chaque batiment elle doit ajouter les effets (effet_argent, effet_pollution…).
Il faut aussi créer une nouvelle fonction pour calculer la population. La fonction doit en gros comparer le nombre de maisons au nombre d'habitants et doit ajuster la population en fonction du bonheur.
Faut aussi ajouter un truc pour les niveaux, genre si on a 50 habitants on passe au niveau 2 pour débloquer les nouveaux bâtiments.

Dans le code Batiment :

Il faut ajouter un attribut self.image dans le init de la classe Batiment pour lier chaque batiment à une image.
Donc il faut avoir trouvé une image pour chaque batiment et faire un dictionnaire qui dit quelle image va avec quel nom.
Il faut aussi vérifier que les prix et les effets de chaque bâtiment sont bien équilibrés pour pas que ce soit trop facile.

Dans le code interface : (Chloé)


Il faut créer une fonction pour les boutons, qui permet ensuite de cliquer dessus et de savoir si on est en mode "placer", "poubelle" ou "info".
Il faut améliorer le dessiner carte pour afficher les vraies images au lieu des carrés bleus.
Et il faut ajouter un compteur de temps pour que actualiser_economie ne s'exécute qu'une fois toutes les deux secondes.
Il faut créer une fenêtre qui s'ouvre pour afficher la boutique (qui contient les bâtiments par catégories).

Dans le code Carte : 

Il faut créer une nouvelle fonction calculer_bonus_proximité, qui en gros prend en paramètre des coordonnées et qui doit par exemple, si on pose une maison à côté d'un parc ça doit donner un bonus de bonheur supplémentaire.
Donc il faut aussi voir quelles conditions donner à quel batiment et créer les fonctions qui vont avec.

Dans le nouveau code Event :

Créer une classe Event.
Créer une fonction générer_evenement qui a une chance sur 100 de se déclencher, tout en affichant un message sur l'interface (genre une canicule) et qui change les indicateurs.
